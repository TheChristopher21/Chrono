# Google Play Data Safety Draft

This is a working draft for the Play Console Data safety form. Final answers must match the deployed backend, privacy policy and production data processing.

## Collection

Chrono collects user data when a user logs in and uses the app with the Chrono backend.

Likely data types:

- Personal info: name, username, email address, personnel number and role information.
- App activity: time tracking punches, daily notes, absence requests, correction requests, customer/project/task assignments and admin workflow actions.
- Financial info: payslip and payroll-related information where enabled for the organization.
- Files and documents: payslip or document metadata where generated by the backend.
- Device or other IDs: authentication/session token stored locally for login persistence.

## Sharing

No third-party advertising or analytics SDK is currently included in the native Android project.

Data is sent to the Chrono backend selected by build type:

- Debug: `http://10.0.2.2:8080`
- Release: `https://api.chrono-logisch.ch`

If the production backend uses subprocessors, hosting providers, email services or support tooling, those must be reflected in the public privacy policy and final Play Console declaration.

## Security Practices

- Network access is required.
- Release builds disallow cleartext traffic.
- Debug cleartext traffic is restricted to emulator/local development hosts.
- Release demo login is disabled.
- Session tokens are stored locally in Android shared preferences.
- Access to app sections is restricted by backend roles and page permissions.

## Data Deletion

Play Console asks whether users can request data deletion. Use the public privacy policy/contact mechanism:

- Privacy policy: https://chrono-logisch.ch/datenschutz
- Contact: info@chrono-logisch.ch

Confirm the exact deletion workflow before production submission.
